## 📖 Overview
This repository contains the complete source code and infrastructure configuration for my implementation of the **Cloud Resume Challenge**. 

Instead of deploying via the AWS Console, this project was built entirely using **Infrastructure as Code (IaC)** with Terraform. The architecture represents a full-stack, globally distributed serverless application, complete with a fully automated CI/CD pipeline.

---

## 🏗️ Architecture

![Cloud Resume Architecture](frontend/assets/images/architecture.svg)

### **How it Works**
1. **Frontend:** The static website (HTML/CSS/JS) is hosted in an **Amazon S3** bucket.
2. **Delivery:** **Amazon CloudFront** serves the website globally via a CDN, secured by an **ACM SSL Certificate** to enforce HTTPS. 
3. **DNS:** Custom domain routing is handled externally via **Cloudflare DNS** (operating in DNS-only mode).
4. **Backend:** When a user visits the page, Javascript triggers an HTTP POST request to an **API Gateway** endpoint.
5. **Compute:** The API Gateway invokes an **AWS Lambda** function written in Python.
6. **Database:** The Lambda function connects to **Amazon DynamoDB**, retrieves the current visitor count, increments it by 1, saves the new value, and returns the data to the frontend to be displayed.

| Category | Technology | Purpose |
|----------|------------|---------|
| **Frontend** | HTML5, CSS3, JavaScript | User interface and client-side API fetching. |
| **Backend** | Python (Boto3) | Serverless compute logic to interact with the database. |
| **Database** | Amazon DynamoDB | NoSQL database to store the persistent visitor count. |
| **Infrastructure**| Terraform | Defines and provisions all AWS resources as code. |
| **CI/CD** | GitHub Actions | Automates the deployment of frontend assets to S3. |
| **Networking** | CloudFront, API Gateway, Route53, Cloudflare | Content delivery, API routing, and DNS management. |

## 🚀 Deployment Pipeline (CI/CD)

This project uses **Continuous Deployment**. Any push to the `main` branch affecting the `frontend/` directory triggers a GitHub Actions workflow that:
1. Provisions an Ubuntu runner.
2. Configures AWS credentials via secure GitHub Secrets.
3. Syncs the updated frontend files to the Amazon S3 bucket (`aws s3 sync`).
4. Creates a CloudFront invalidation to instantly push the new cached assets to edge locations globally.

### Required GitHub Secrets:
* `AWS_ACCESS_KEY_ID`
* `AWS_SECRET_ACCESS_KEY`
* `AWS_S3_BUCKET`
* `AWS_CLOUDFRONT_ID`
